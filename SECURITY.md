# Security Policy

## Reporting a vulnerability

If you discover a security issue, please report it responsibly.

### Preferred way
- Open a private report or email the maintainer directly
- Include reproduction steps or a proof-of-concept if possible

### What to include
- Description of the issue
- Affected version / commit
- Potential impact
- Suggested fix (if available)

## Scope

This project is focused on Skill lifecycle tooling and local maintainer workflows.
Please avoid actions that would negatively affect other users or production systems.

## Response

Maintainers will review reports as quickly as possible and work on a fix or mitigation.
