Primary maintainer

## Why this project matters
AI Agent ecosystems are quickly moving from single-prompt usage to long-lived operational systems.
As agents become more capable, one of the biggest hidden bottlenecks is not model performance — it is **maintenance discipline**.

This project focuses on a very specific but important problem:
**How to manage Skills, prompts, SOPs, and reusable workflows as maintainable assets instead of scattered artifacts.**

This matters because many Agent projects now face the same pain:
- Skills accumulate quickly but quality becomes inconsistent
- Pitfalls are repeated across sessions and contributors
- Maintenance burden grows faster than capability
- There is no standard way to validate reusable workflow assets
- Teams struggle to turn experience into long-term operational knowledge

Agent Skill Lifecycle Kit is designed to help open source maintainers solve this class of problems.

## How Codex tools and API credits will be used
If selected, Codex and API credits will be used directly in maintenance workflows:

1. **Issue triage**
   - Classify incoming issues faster
   - Identify which reports relate to templates, checks, CLI behavior, or adapter compatibility

2. **PR review**
   - Review changes to Skill validation logic
   - Check documentation quality
   - Evaluate impact on existing templates and contributor workflows

3. **Release automation**
   - Improve versioning and packaging workflows
   - Maintain release notes
   - Reduce manual overhead for small but frequent releases

4. **Documentation improvement**
   - Help improve contributor guides
   - Improve onboarding documentation for real maintainer workflows
   - Keep docs aligned with evolving project structure

5. **Adapter development**
   - Improve compatibility with popular Agent systems such as Hermes, OpenClaw, Codex, and Claude Code
   - Translate practical maintenance patterns into reusable project features

## Expected impact on maintenance burden
This project is intentionally positioned at the maintenance layer rather than the model layer.
The expected impact is:

- Better consistency across Skill assets
- Faster detection of incomplete or low-quality Skills
- Lower onboarding cost for new contributors
- More structured handling of Pitfalls and reusable SOPs
- Clearer audit trail for reusable workflow assets
- Reduced long-term maintenance overhead for Agent teams

## Why this maintainer should be selected
This maintainer is focused on practical infrastructure for Agent operations, not just experimental prompts or one-off automations.
The project is being built around real maintainer problems:

- maintainability
- reusability
- governance
- contributor clarity
- long-term operational sustainability

This aligns well with the goals of Codex for OSS, because the project is intended to help open source maintainers reduce real coding and review burden while improving the quality of reusable workflow assets.
