# Maintenance Guide

This document describes how the project is maintained.

## Maintainer responsibilities

- Review issues and pull requests
- Keep documentation accurate
- Improve skill quality rules
- Avoid breaking existing workflows unless necessary
- Keep templates practical and realistic

## Release process

1. Update version in code / metadata
2. Update docs
3. Tag release
4. Publish notes

## What makes a good change

- Solves a real maintainer problem
- Keeps CLI ergonomics simple
- Prefer incremental improvements over large speculative changes
- Improves clarity for new contributors

## Long-term direction

This project should remain focused on:

- Skill lifecycle management
- Maintenance workflow automation
- Agent asset governance
- Practical tooling over theory
