# Codex for OSS Application Draft (for OpenAI submission)

## Project name
Agent Skill Lifecycle Kit (ask)

## GitHub repository
https://github.com/chenguagua/ask

## Role
Primary maintainer

## Why this project matters
AI Agent ecosystems are moving fast from single-prompt usage to long-lived operational workflows.
One of the biggest hidden maintenance burdens is **Skill lifecycle management**:
how teams create, validate, evolve, retire, and audit Skills, prompts, SOPs, and reusable workflows.

This project matters because it targets a concrete bottleneck for Agent maintainers:
- Skills accumulate quickly
- Quality becomes inconsistent
- Pitfalls are repeated across sessions
- Knowledge is scattered across chat history, docs, and private notes
- Maintenance cost grows faster than capability

Agent Skill Lifecycle Kit helps maintainers treat Skills as **maintainable software artifacts**, not throwaway prompt snippets.

## How API credits and Codex tools will be used
The maintainer will use Codex and API credits to:
1. Accelerate PR review and issue triage for Skill schema changes
2. Improve automated validation rules for real-world Agent workflows
3. Build adapters for popular Agent stacks (e.g. Hermes / OpenClaw / Codex / Claude Code)
4. Maintain release automation for templates, checks, and exporters
5. Improve documentation and contributor onboarding

## Expected impact on maintenance burden
- Fewer inconsistent Skills in production workflows
- Faster detection of incomplete or fragile Skills
- Better onboarding for contributors
- Clearer audit trail for reusable workflow assets
- Lower long-term maintenance overhead for Agent teams

## Why the maintainer should be selected
This maintainer is actively building practical Agent operations infrastructure, not just publishing experimental prompts.
The project is intentionally focused on maintainability and long-term operator workflows, which aligns well with the goals of Codex for OSS.
